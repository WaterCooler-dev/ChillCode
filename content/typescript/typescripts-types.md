# 타입스크립트의 타입들

타입스크립트를 시작할때, 제가 이렇게 말했습니다.

> "타입스크립트는 **변수에 타입을 지정**할 수 있습니다."

그럼 일단은 타입스크립트에서는 변수에 타입을 지정할 수 있다는 것은 압니다. 그럼, 어떤 타입을 지정할 수 있을까요?

## 타입스크립트의 타입 특징

타입스크립트의 타입은 다른 언어들과는 다릅니다. 다른 언어, 예를 들어 자바에서는 타입을 표현할 때 이런 이름 (int, double등...)을 씁니다. 다음은 자바에 있는 여러 타입의 예시입니다.

```java
int 숫자 = 12345;
double 소수 = 3.141592653;
String 문자 = "다람쥐 헌 맥북이라도 사고파";
boolean 불린 = false;
```

하지만 타입스크립트에서는 다르게 표현합니다. 저 예시를 타입스크립트로 바꾸면...

```typescript
let 숫자: number = 12345; // 자바의 int
let 소수: number = 3.141592653; // 자바의 double
// 모든 수를 number로 표현함
let 문자열: string = "다람쥐 헌 맥북이라도 사고파"; // 자바의 string
let 불린: boolean = false; // 자바의 boolean
```

이렇게 표현됩니다. 특히 소수와 정수가 하나의 타입 `number`로 합쳐져 있는게 인상깊습니다. 이렇듯 다른 언어와 달리 타입스크립트의 타입은 살짝 다릅니다.

## 타입스크립트의 원시 타입들

::: tip
String, Number, Boolean 같은 앞이 대문자인 타입들도 있고, 이것들을 변수의 타입으로 지정해도 코드는 잘 작동 하지만 이 타입들은 **래퍼 객체** 라고 불리는 클래스들이며, 사용했을때 불안전하고 예측 불가한 경우가 많으니 **최대한** 사용하지 말길 바랍니다. (하지만 아주 간혹가다 사용할 때도 있긴 합니다...)
:::

### number

다른 언어의 int, double, float... 같은 **숫자 타입**에 대응하는 원시 타입입니다. 1, 2, 3.141592같은 숫자 값을 보관 할 수 있습니다.

```typescript
let pi: number = 3.141592653589;
```

### string

다른 언어의 string같은 **문자열 타입**에 대응하는 원시 타입입니다. "간장공장공장장은 강 공장장이고, 된장 공장 공장장은 공 공장장이다.", "다람쥐 헌 맥북이라도 사고파" 같은 문자열 값을 보관 할 수 있습니다.

```typescript
let squirrelHasNoMoney: string = "다람쥐 헌 맥북이라도 사고파";
```

### boolean

다른 언어의 boolean같은 **불린 타입**(참, 거짓)에 대응하는 원시타입입니다. true(참), false(거짓) 같은 불린 값을 보관 할 수 있습니다.

```typescript
let isThisBlogsMasterHandsome: boolean = false; // ㅠ
```

## 타입스크립트의 배열 타입

자바스크립트에서 배열을 사용하시는 방법은 아실겁니다.

```typescript
let array = ["1", "2", "3"];
```

근데 이 배열에 어떤 값이 들어갈수 있는지 알 수 있으면 더욱 좋지 않을까요? 그래서 타입스크립트는 배열을 위한 타입을 제공합니다.

타입스크립트에서 배열을 위한 타입을 사용하기 위해서는 `타입[]`을 사용할 수 있습니다.

```typescript
let numbers: number[] = [1, 2, 3.141592];
```

아니면 `Array<타입>`을 사용할 수도 있습니다.

```typescript
let numbers: Array<number> = [1, 2, 3.141592];
```

## 특별한 타입 any

타입스크립트에서는 타입 관련해서 오류가 발생하지 않기를 원할 수 있습니다. 예를 들자면 이런 상황에서요:

```typescript
let response: ??? = JSON.parse('{"칠코드":"좋음👍👍👍👍"}'); 
// ???에 타입으로 무엇을 넣어야 하나요? 잘못 넣으면 오류가 날 겁니다.
```

이 코드는 문자열을 JSON(오브젝트)로 변환해주는 코드입니다. 근데, 만약 매개변수 안 문자열이 어떤 내용인지 모른다면, 타입은 무엇으로 지정해야 할까요? 이럴때 any를 사용합니다.

any는 어떤 값이든 담게 할 수 있습니다. 자바스크립트의 일반적인 변수와 같습니다. 문자열을 넣었다, 숫자를 넣었다 할 수 있고, 어떤 값이 들어가 있는지 알 수 없죠. 하지만 타입스크립트에서는 이 아무 값이나 넣을 수 있다는 것이 장점이 될 수도 있습니다.

앞에 코드를 any 타입으로 바꿔봅시다.

```typescript
let response: any = JSON.parse('{"칠코드":"좋음👍👍👍👍"}');
// 이제 타입 걱정을 할 필요는 없군요?
```

이렇게 하면 어떤 타입을 써야 할지 모를때, 타입이 잘못 돼 오류가 날 걱정은 없을 겁니다.
하지만 이 any 타입을 남용하진 마세요. 아무 타입이나 들어 갈 수 있기 때문에, 오류가 생길 확률이 높아집니다. 차라리 그럴거라면 솔직히 자바스크립트를 하는게 훨씬 낫습니다.
